## <u>Teamate</u> Project Background
- ### 필요성
  - 운전 중 스마트폰 조작은 사고 위험을 높이며, 음성 명령을 통한 차량 제어 필요성이 증가하고 있음
  - 차량 내부의 공기질(특히 CO₂ 농도)에 따라 졸음운전 위험이 높아질 수 있음
  - 운전자 중심의 맞춤형 음성 서비스와 실내 환경 관리 시스템 필요
  - 기존 상담원 기반 서비스를 AI로 대체하여 비용 절감 및 즉각적인 응답 제공
  - 차량에 대해 잘 모르는 일반 운전자는 차량 정보 등을 알고 싶을 때,
    두꺼운 종이 설명서나 인터넷 검색을 통해 직접 찾아야 하는 번거로움을 겪음
  - 따라서 운전자가 AI 비서에게 음성으로 질문하여 차량 정보를 즉시 안내받을 수 있는 시스템을 제공
  - 추후 차량 제조사와 관계없이 사용가능한 범용적인 AI 비서 시스템 구축
    
- ### 기존 해결책의 문제점
  - 차량 매뉴얼이나 인터넷 검색은 운전 중 접근이 어렵고, 차량 모델별 정보가 다를 수 있어 찾기 번거로움
  - 차량 제조사별 전용 AI 시스템은 특정 차량 모델에만 적용되어 범용성이 부족함
  - 고객센터 상담원 기반 시스템은 운영 비용이 높고, 응답 속도가 느려 즉각적인 대응이 어려움
  - 커스터마이징이 불가능하고, 기능 확장성 부족
  - 기존 음성 서비스는 공기 같은 차량 내부 환경 데이터를 활용하지 못해, 운전자 안전 관리 기능이 제한적임
  
## System Design
  - ### System Requirements
    - Raspberry Pi 5 기반 제어 보드
    - 음성 인식 모듈 및 미세먼 센서 연결
    - React + Tailwind CSS 기반의 사용자 UI (LCD 패널 출력)
    - 데이터베이스를 활용해 차종별 질문 저장
    - OpenAI API를 활용한 자연어 처리 및 응답 생성
    
## Case Study
  - ### Description
    - 운전자가 "에어컨 필터에서 이상한 냄새가 나는 것 같아" 등 음성 질문을 하면,
      -> 기존 데이터베이스에 저장된 질문이면 그 답변을 출력
      -> 저장되지 않은 질문이면 OpenAI API를 이용한 답변을 출력 후 데이터베이스에 저장
      -> LCD 패널에 답변 출력 및 음성 제공
    - 실시간 공기질 농도 측정 기능
      -> 공기질이 나빠졌을 경우 경고 메시지 및 음성 알림으로 환기 유도
  
  
## Conclusion
  - ### 차량용 AI 비서 시스템으로, 차량 내 편의성과 안전성 향상
  - ### 추후 차량 데이터 연동 및 AI 기능을 이용해 서비스 확장 가능 또는 차량용 IoT 플랫폼, 카셰어링 서비스 등과 연동 가능
  - ### 추후 공기질 센서를 활용해 유해 가스를 감지하고, 위험 상황 발생 시 외부로 알림을 전송하는 등 안전 보조 기능으로 확장 가능
